# Contributing to twoChurch

We welcome contributions to twoChurch! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/ITSolutionsLLC/twoChurch.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Commit your changes: `git commit -m "Add your message"`
6. Push to your fork: `git push origin feature/your-feature-name`
7. Create a Pull Request

## Development Setup

```bash
# Install dependencies
npm install

# Run web app
npm run web

# Run mobile app
npm run mobile
```

## Code Style

- Use TypeScript for all new code
- Follow existing code style
- Run linter before committing: `npm run lint`
- Write meaningful commit messages

## Pull Request Process

1. Update the README.md with details of changes if needed
2. Update the version numbers following [SemVer](http://semver.org/)
3. The PR will be merged once you have approval from maintainers

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Focus on what is best for the community
